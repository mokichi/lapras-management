# LAPRAS 職務経歴書ビルダー

## 概要
提供されたツールを介して LAPRAS から利用可能な情報を集約し、ユーザーから提供された基本情報や学歴を組み合わせて、詳細な Markdown 形式の職務経歴書 (`outputs/resume.md`) を生成します。

## ワークフロー

### 1. データ取得
以下のデータを並列で取得します：
- `get_job_summary()`: 職務要約テキストを取得します。
- `get_experiences()`: 完全な職務経歴リストを取得します。
- `get_tech_skill()`: テクニカルスキルと経験年数のリストを取得します。
- `get_want_to_do()`: キャリアの志向性や「今後やりたいこと」を取得します。

### 2. 経歴の処理（ソート）
`get_experiences()` から取得した `experience_list` を以下の優先順位でソートします：
1.  **進行中のプロジェクトを優先**: `end_year == 0`（またはレスポンス内の0）のアイテムを最上位にします。
2.  **時系列順（降順）**: 進行中および終了済みの両方のグループ内で、`start_year`（降順）、次に `start_month`（降順）でソートします。

### 3. 出力要件
- **省略なし**: `experience_list` にあるプロジェクトを要約、グループ化、または省略しないでください。すべての項目を含める必要があります。
- **詳細な説明**: ツールから提供された各経歴の完全な `description` を含めてください。
- **詳細な説明**: `description` は以下の条件に従ってください。
  - `title` は含めないでください。
  - 見出しはレベル4から始めてください。
  - 内容は省略しないでください。
- **テンプレートの使用**: `assets/resume_template.md` にある構造をガイドとして使用してください。

## テンプレートガイド
基本構造については `assets/resume_template.md` を参照してください。

### スキルパラメータ
- `today`: 「YYYY年MM月DD日」形式でフォーマットします。
- `tech_skills_list`: 「スキル（年数）」を示す箇条書きリストまたは表形式でフォーマットします。
- `start_date` / `end_date`: 「YYYY年MM月」形式でフォーマットします。`end_year == 0` の場合は「現在」を使用します。
- `roles`: すべての `job_position_name` の値をカンマで連結します。
